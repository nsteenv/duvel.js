Application web avec une grosse base de code

De plus en plus d'erreurs à chaque release

Besoin de tester le font-end!

Comment faire?

Faire des tests fonctionnels?

Ecrire des tests pour toute l'application?

Surtout: personne n'avait envie de le faire!

Dans ces moments là...

Tu peux pas test!

Solution: regression testing

(un peu brutale)

Et si on fesait des screenshots de toute l'appli?

En comparant les différences entre les révisions?

Et en automatisant ce processus?

Par exemple: deux screenshots

On compare les différences

Permet de détecter

Les changements de styles, layout, couleurs (même minimes)

Les changements de contenu

Les problèmes d'affichage entre différentes résolutions

Ne permet pas de détecter

La cohérence des données

La notion d'erreur

Demande un jugement humain

Départager les changements voulus des erreurs

Avoir une interface pour valider le tout facilement

Conclusion

On gagne énormément de temps!

Ex: 5-10mn pour valider ~300 screenshots

S'adapte à n'importe quel langage/architecture web

Pas besoin de connaissance spécifique, il suffit de connaîte son application

Permet de détecter des erreurs qu'on ne verrait pas avec des tests fonctionnels

Ressources

Duvel.js

Crawler JS permettant de capturer les screenshots du site

Imagemagick

Boite à outils de manipulation d'images. Permet de faire des comparaisons/différences entre deux images.
